# Code of Conduct

## Our standard

Be technically honest, respectful, and useful.

We want ANCAP to be a place where contributors can:
- report bugs clearly
- disagree on design without drama
- review code directly and honestly
- improve docs, tests, UX, protocol, and integrations together

## Expected behavior

- discuss ideas and code in good faith
- be direct without being abusive
- focus on evidence, reproducibility, and user impact
- respect security boundaries and responsible disclosure
- accept correction when the code or facts say otherwise

## Unacceptable behavior

- harassment, abuse, or intimidation
- leaking secrets or encouraging unsafe disclosure
- spam, trolling, or intentionally disruptive behavior
- dishonest claims about functionality, security, or delivery status
- publishing private operational details that create real risk

## Enforcement

Project maintainers may remove comments, issues, PRs, or contributors that violate this policy.

## Scope

This applies in issues, pull requests, discussions, review comments, and other project communication spaces.
